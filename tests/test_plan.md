# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] El programa corre al menos una vez con entrada válida.
- [ ] Se probó al menos un caso de entrada inválida.
- [ ] Se verificó comportamiento en un caso límite simple.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` está completo.
- [ ] `docs/caso_de_uso.md` describe flujo principal y alternativo.
- [ ] `docs/plan_de_pruebas.md` tiene casos y resultados.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` ejecuta o guía la ejecución del proyecto.
- [ ] `src/main.<ext>` existe y corresponde al lenguaje elegido.
- [ ] No se usan APIs externas ni servicios de red.

## 4) Checklist de entrega final
- [ ] El alcance sigue siendo pequeño y viable.
- [ ] La estructura del repositorio se respeta.
- [ ] Los archivos tienen nombres claros y consistentes.
- [ ] El repositorio está listo para revisión en GitHub Classroom.
